v1.3.1 / Build 7 / 2026-06-30

新增:
- 菜单栏图标菜单新增“重启本地采样器”，用户遇到连接失败时可以直接手动恢复，无需查端口或重启整台电脑。

改进:
- App 启动后台采样器前会清理同一端口上残留的旧版采样器进程，避免旧进程占用 8765 导致新版连接失败。
- 原生 App 壳新增健康检查: 每 15 秒检查一次本地采样器，连续 3 次无响应会自动重启后台服务。
- 启动等待逻辑现在会校验 /api/state 的 JSON 内容，避免把空响应或异常响应误判为启动成功。
- 打包脚本在重新构建前会先尝试停止旧版 App 和旧采样器，减少“运行中旧进程 + 已覆盖新版资源”的错位状态。

v1.1.2 / Build 4 / 2026-06-30

修复:
- 修复 App 长时间运行或频繁刷新后可能显示“连接本地采样器失败: Load failed”的问题。
- 修复后台 Python 服务输出管道无人读取时可能被 stderr/stdout 堵塞，导致本地 API 不响应的问题。
- 后台服务现在会静默处理浏览器断连产生的 BrokenPipe/ConnectionReset 类错误。
- 前端轮询偶发失败时会保留上次成功数据并自动重试，连续失败后才显示红色错误。

说明:
- 这次问题不是采样器数据丢失，而是本地 HTTP 接口线程被输出管道堵住；新版会持续清空后台输出并避免断连堆栈刷屏。

v1.1.0 / Build 2 / 2026-06-30

新增:
- 新增 macOS 菜单栏状态图标。
- 新增后台运行模式: 点击窗口关闭按钮时只隐藏窗口，监控继续运行。
- 新增菜单栏图标菜单项: 显示硬盘读写监控、隐藏窗口、退出硬盘读写监控。
- 新增退出保护: 普通退出事件会被拦截，避免误关后台监控；真正退出需要从菜单栏图标菜单选择“退出硬盘读写监控”。
- 新增“休眠期间继续监控”开关。
- 新增日志文件夹选择功能，并记住用户上次选择的日志保存位置。
- 新增 VERSION.txt 和 CHANGELOG.txt，用于记录版本号和更新日志。

改进:
- 修改采样间隔或日志间隔时，不再重置当前已经排定的下一次日志保存时间。
- 关闭“休眠期间继续监控”时，系统睡眠/唤醒之间会暂停日志倒计时，避免醒来后立即补写日志。
- 开启“休眠期间继续监控”时，App 会向 macOS 申请防止空闲睡眠，以便持续采样。
- 日志默认保存到 ~/Library/Application Support/硬盘读写监控/logs。
- 用户选择的日志目录会保存到 ~/Library/Application Support/硬盘读写监控/config.json。
- App 包构建时会进行本机 ad-hoc 签名。

说明:
- “休眠期间继续监控”主要防止空闲自动睡眠；手动睡眠、合盖或系统强制睡眠仍可能暂停 App。
- 当前版本可以按应用/进程统计磁盘读取、写入和实时速率；暂不追踪具体写入了哪些文件。

v1.0.0 / Build 1 / 2026-06-30

新增:
- 初始版本。
- 基于 macOS proc_pid_rusage 读取进程级磁盘读写累计数据。
- 提供本地 Web GUI，按应用/进程聚合展示读写量。
- 支持启动后读取、启动后写入、当前读速、当前写速展示。
- 支持按应用、进程路径和 PID 筛选。
- 支持手动保存 TXT 日志。
- 支持默认每 30 分钟自动保存 TXT 日志。
- 支持打包为 macOS .app，双击启动。
- App 内置 Python 后台采样服务和网页界面。
