这是一个 macOS 本地硬盘读写监控工具，用系统底层 `proc_pid_rusage` 接口读取每个 PID 的累计磁盘读取/写入字节数，并在本地网页界面中按应用或进程聚合展示。

## 启动

```bash
./start.sh
```

然后打开：

```text
http://127.0.0.1:8765/
```

## 打包成 Mac App

```bash
./build_mac_app.sh
open "dist/硬盘读写监控.app"
```

打包后的 App 会把监控服务和网页界面都放进 `.app` 内部。双击 App 后会打开一个原生 macOS 窗口，退出 App 时后台采样服务也会一起停止。

通过 App 启动时，TXT 日志会保存到：

```text
~/Library/Application Support/硬盘读写监控/logs
```

如果你在 App 里选择了别的日志文件夹，选择结果会保存到：

```text
~/Library/Application Support/硬盘读写监控/config.json
```

下次启动 App 时会继续使用你上次选择的日志文件夹。

关闭窗口只会把 App 隐藏到后台，监控仍会继续运行。需要彻底退出时，请在菜单栏图标中选择“退出硬盘读写监控”。

## 功能

- 按应用/进程展示启动本工具后的读取、写入增量。
- 展示当前读写速度，速度口径为最近一次真实采样周期内的读写速率。
- 展示进程自启动以来的累计写入，辅助判断老进程历史 IO。
- 默认每 30 分钟写入一份 TXT 日志到 `logs/`。
- 支持手动保存当前日志。
- 支持在 App 内选择 TXT 日志保存文件夹。
- 软件页面不再展示 TXT 日志列表，避免主界面过长；日志仍保存到所选文件夹。
- 正常退出或收到可拦截终止信号时，会尽力保存一份关闭快照日志。
- 支持选择休眠期间是否继续监控。
- 支持筛选应用、进程路径和 PID。
- 支持点击应用/进程名查看完整可执行路径，并复制路径或完整信息。
- 支持在进程详情中隐藏某个应用/进程的前台显示；隐藏不影响采样、TXT 日志、日报或周报。
- 右上角设置按钮可查看隐藏项列表，并恢复特定应用/进程显示。
- 进程排名表头提供明确的升序/降序排序图标按钮。
- 支持自定义读写合计阈值提醒，并通过右上角消息图标查看提醒历史。
- 消息图标会显示提醒数量角标，点击后用浮层打开可滚动的消息中心。
- 支持按用户设置的时间自动生成日报和周报 TXT 汇总；日报和周报都以用户设置的汇报时刻作为统计周期边界。
- 日报、周报设置旁提供统计口径提示，可查看当前周期起点和下一次写入时间。
- 内置本地采样器自修复: 启动时清理旧采样器，运行中连续无响应会自动重启。
- 菜单栏图标中提供“重启本地采样器”，遇到连接失败时可手动恢复。
- 菜单栏图标带状态灯: 绿色表示采样器正常，红色表示采样器无响应或报错。
- 右键菜单栏图标可打开状态浮窗，查看启动后写入、启动后读取、当前写速和下次日志剩余时间。
- Mac App 会自动避让本地端口冲突: 优先使用 8765，如果被其他软件占用，会自动改用备用端口。

## 说明

这个版本可以量化“哪个进程写入了多少字节”。它不会列出每个进程具体写了哪些文件；macOS 要做到文件路径级追踪通常需要 `fs_usage`/DTrace 权限，也更适合做临时诊断而不是长期后台监控。

修改采样间隔或日志间隔不会刷新当前已经排定的下一次日志保存时间。新的日志间隔会在下一次日志写完之后再用于计算后续保存时间。

TXT 日志会记录真实的统计区间起止时间和统计时长；如果在某段统计中途修改日志间隔，已经经过的这一段仍按真实时间写入日志。

关闭快照日志同样使用真实时间: 统计区间从上一份日志写入时间到关闭瞬间，不会按原本设定的日志间隔虚填时长。断电、系统崩溃或 kill -9 这类不可拦截终止无法保证写入最后日志。

“休眠期间继续监控”开启时，App 会向 macOS 申请防止空闲睡眠，以便继续采样和写日志。关闭时，电脑可以正常睡眠；App 会在睡眠/唤醒之间暂停日志倒计时，避免醒来后因为睡眠时间经过而立即补写一条日志。

如果界面显示“连接本地采样器失败”，通常等待几十秒会由 App 自动重启采样器。也可以右键菜单栏图标，选择“重启本地采样器”立即修复。

Mac App 启动时只会清理本软件残留的旧采样器进程；如果 8765 被其他本地开发软件占用，会自动换到多个备用端口段中的空闲端口，不会关闭其他软件。
